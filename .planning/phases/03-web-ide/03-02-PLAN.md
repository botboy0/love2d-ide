<task type="auto">
  <name>Task 1: Process manager, watcher, console SSE, and run/stop routes</name>
  <files>ide/lib/process-manager.js, ide/lib/watcher.js, ide/routes/run.js, ide/routes/console.js, ide/server.js</files>
  <action>
1. Create `ide/lib/process-manager.js`:
   - Import `spawn` from `node:child_process`, `consoleBus` from `./console-bus.js`
   - Module-level `let currentProcess = null`
   - `killCurrent()`: If `currentProcess` exists, call `currentProcess.kill()` (no signal argument — Windows maps to TerminateProcess), set to null. Return immediately if no process.
   - `launch(projectPath, loveExePath)`: Call `killCurrent()` first. Emit `consoleBus.emit('clear')`. Spawn: `spawn(loveExePath, [projectPath], { shell: false, windowsHide: false })`. Wire `stdout.on('data')` and `stderr.on('data')` to emit `consoleBus.emit('line', {stream, text})`. Wire `close` event to emit info line with exit code and set `currentProcess = null`.
   - `isRunning()`: Returns `currentProcess !== null`
   - Export all three functions.

2. Create `ide/lib/watcher.js`:
   - Import chokidar, `launch` and `killCurrent` from `./process-manager.js`
   - `startWatcher(projectPath, loveExePath)`: Create chokidar watcher on `projectPath` with `usePolling: false` (Windows NTFS native events), `ignoreInitial: true`, `ignored: /(^|[\/\\])\../`. On `change` event, filter to `.lua` files only. Debounce 350ms — on debounced change, call `launch(projectPath, loveExePath)` to restart the game. Store watcher reference.
   - `stopWatcher()`: Close the watcher if it exists.
   - Export both functions.
   - **Important:** chokidar v4 does NOT support glob patterns in watch path. Pass the directory path string directly.

3. Create `ide/routes/run.js`: Express Router with:
   - `POST /` — Reads `lovePath` and `projectPath` from config (passed via closure or req.app.locals). Calls `launch(projectPath, lovePath)`. Returns `{ status: 'running' }`.
   - `POST /stop` — Calls `killCurrent()`. Returns `{ status: 'stopped' }`.

4. Create `ide/routes/console.js`:
   - `consoleStream(req, res)`: SSE endpoint. Set headers: `Content-Type: text/event-stream`, `Cache-Control: no-cache`, `Connection: keep-alive`. Call `res.flushHeaders()`.
   - Register listener on `consoleBus` for `line` event — write `data: ${JSON.stringify(lineData)}\n\n` to response.
   - Register listener for `clear` event — write `event: clear\ndata: {}\n\n`.
   - **Critical:** Register `req.on('close', cleanup)` immediately after flushHeaders to remove listeners. Use the SAME function reference for on/off to prevent SSE connection leaks.

5. Update `ide/server.js`:
   - Import and mount `runRouter` at `/api/run`
   - Mount `consoleStream` at `GET /api/console/stream`
   - Import `startWatcher` and call it on server startup with `config.projectPath` and `config.lovePath`
   - Store config values in `app.locals` for route access
  </action>

<task type="auto">
  <name>Task 2: Wire frontend Run/Stop buttons, console panel, error links, and save-triggered reload</name>
  <files>ide/public/app.js, ide/public/style.css</files>
  <action>
1. Update `ide/public/app.js` — replace stub handlers with real implementations:

   **Run button:**
   - On click, `fetch('/api/run', { method: 'POST' })`. Update status indicator to "Running" (green dot or text). Disable Run button while running (optional UX).

   **Stop button:**
   - On click, `fetch('/api/run/stop', { method: 'POST' })`. Update status indicator to "Stopped".

   **Console panel SSE:**
   - On page load, create `new EventSource('/api/console/stream')`.
   - On `message` event: Parse JSON data (`{stream, text}`). Create a `<div>` element with class based on stream type (`console-stdout`, `console-stderr`, `console-info`). If `stream === 'stderr'`, run error linkification on the text before inserting.
   - On `clear` custom event: `es.addEventListener('clear', () => consoleOutput.innerHTML = '')`.
   - Auto-scroll: After each append, `consoleOutput.scrollTop = consoleOutput.scrollHeight`.

   **Error linkification:**
   - Regex: `/([^:\s]+\.lua):(\d+)/g`
   - Replace matches with `<a href="#" data-file="$1" data-line="$2" class="error-link">$1:$2</a>`
   - Delegate click on `#console-output` for `.error-link` clicks:
     - `e.preventDefault()`
     - Extract `data-file` and `data-line`
     - Fetch the file content: `GET /api/files/${file}`
     - Load into editor: `openFile(content)` (or reuse existing if same file)
     - Navigate: `goToLine(parseInt(line))`

   **Save-triggered reload:**
   - In the existing Ctrl+S handler, after the PUT request succeeds, also call `fetch('/api/run', { method: 'POST' })` to trigger immediate restart. This gives instant feedback without waiting for chokidar's 350ms debounce.

2. Update `ide/public/style.css` — add console output styles:
   - `.console-stdout`: color Catppuccin Text (`#cdd6f4` dark / `#4c4f69` light)
   - `.console-stderr`: color Catppuccin Red (`#f38ba8` dark / `#d20f39` light)
   - `.console-info`: color Catppuccin Overlay0 (`#6c7086` dark / `#9ca0b0` light), italic
   - `.error-link`: color Catppuccin Blue (`#89b4fa`), underline, cursor pointer
   - Console output container: `font-family: 'Cascadia Code', 'Fira Code', monospace; font-size: 13px; padding: 8px; overflow-y: auto`

**Important details:**
- SSE `EventSource` automatically reconnects on disconnect — no manual reconnect logic needed
- Error links must work for both relative paths (`main.lua`) and nested paths (`src/player.lua`)
- Save-triggered reload should only fire if a game process is currently running (check status)
