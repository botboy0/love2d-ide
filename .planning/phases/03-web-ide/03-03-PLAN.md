1. Create `ide/routes/export.js`: Express Router with:
   - `GET /` — Streams a ZIP archive of the project directory as a `.love` file download.
   - Read `projectPath` from `req.app.locals.config.projectPath`.
   - Derive project name from the last path segment of projectPath (e.g., `01-pong`).
   - Load `.loveignore` from project root if it exists. Use `ignore` npm package to parse it.
   - Set response headers: `Content-Type: application/zip`, `Content-Disposition: attachment; filename="${projectName}.love"`.
   - Use `archiver('zip', { zlib: { level: 9 } })` to create the archive.
   - Pipe archive to response: `archive.pipe(res)`.
   - Use `archive.glob('**', { cwd: projectPath, dot: false, ignore: [patterns from .loveignore] })` or filter with the `ignore` package.
   - Call `archive.finalize()`.
   - Handle errors: `archive.on('error', err => res.status(500).json({ error: err.message }))`.

<task type="auto">
  <name>Task 2: LSP WebSocket proxy and Monaco language client</name>
  <files>ide/routes/lsp.js, ide/build/editor-entry.mjs, ide/public/editor.bundle.js, ide/server.js, ide/public/app.js</files>
  <action>
1. Create `ide/routes/lsp.js`:
   - Import `WebSocketServer` from `ws`.
   - Export `attachLspProxy(httpServer, lsPath)`:
     - If `lsPath` is empty or falsy, log a warning and return (LSP disabled — no lua-language-server configured).
     - Create `new WebSocketServer({ server: httpServer, path: '/lsp' })`.
     - On `connection`: Spawn `lsPath` (the lua-language-server.exe binary) as a child process with stdio: `['pipe', 'pipe', 'pipe']`.
     - Relay messages between WebSocket and child process stdio:
       - WebSocket message -> write to child process stdin
       - Child process stdout -> send to WebSocket
     - Use `vscode-ws-jsonrpc` for proper LSP Content-Length framing: import `createServerProcess` and `forward` from `vscode-ws-jsonrpc/server`. Create the server process and forward between the WebSocket and process.
     - On WebSocket close: kill the child process.
     - On child process exit: close the WebSocket.

2. Update `ide/build/editor-entry.mjs`:
   - Add imports: `MonacoLanguageClient`, `CloseAction`, `ErrorAction` from `monaco-languageclient`, and `toSocket`, `WebSocketMessageReader`, `WebSocketMessageWriter` from `vscode-ws-jsonrpc`.
   - Add `connectLsp(wsUrl)` function:
     - Creates a WebSocket to `wsUrl`
     - On open: wraps with `toSocket()`, creates reader/writer, creates `MonacoLanguageClient` with `documentSelector: [{ language: 'lua' }]`, starts the client.
     - On reader close: stops the client.
   - Export `connectLsp` alongside `monaco`.

3. Rebuild the Monaco bundle:
   ```
   powershell.exe -Command "cd 'C:\Users\Trynda\Desktop\Dev\Lua\ide'; node build/build.mjs"
   ```

4. Update `ide/server.js`:
   - Import `attachLspProxy` from `./routes/lsp.js`.
   - After `httpServer = app.listen(...)`, call `attachLspProxy(httpServer, config.lsPath)`.
   - The httpServer reference is needed — change from `app.listen()` to `http.createServer(app)` + `httpServer.listen()` if not already done.

5. Update `ide/public/app.js`:
   - After Monaco editor is created, call `window.MonacoIDE.connectLsp('ws://' + location.host + '/lsp')`.
   - This is a one-liner — the bundle handles the rest.

**Important notes:**
- If `config.lsPath` is empty, LSP is gracefully disabled. The IDE still works for editing, running, etc. LSP is an enhancement.
- `vscode-ws-jsonrpc/server` may need to be imported differently depending on the installed version. Check if it's `vscode-ws-jsonrpc/lib/server` or similar. The research suggests `vscode-ws-jsonrpc` v3.x.
- The user must download lua-language-server Windows binary separately and set the path in config.json. This is captured in `user_setup` frontmatter.
  </action>
