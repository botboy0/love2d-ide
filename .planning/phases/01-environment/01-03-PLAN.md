<task type="auto">
  <name>Task 1: Configure ADB alias and package test game for Android</name>
  <files>~/.zshrc</files>
  <action>
1. Check if ADB is already accessible on Windows. Try:
   ```bash
   /mnt/c/platform-tools/adb.exe version 2>/dev/null || echo "NOT_AT_DEFAULT"
   cmd.exe /C "adb version" 2>/dev/null || echo "NOT_ON_PATH"
   ```
   Also check common alternate locations:
   - /mnt/c/Users/Trynda/AppData/Local/Android/Sdk/platform-tools/adb.exe
   - /mnt/c/Android/platform-tools/adb.exe

   If NOT found anywhere, create a checkpoint asking the user to:
   a. Download platform-tools from https://developer.android.com/tools/releases/platform-tools
   b. Extract to C:\platform-tools\
   c. Add C:\platform-tools\ to Windows System PATH
   d. Open a new terminal and verify `adb version` works in PowerShell

2. Once adb.exe is found, add alias to ~/.zshrc (append, check for existing alias first):
   ```bash
   alias adb='/mnt/c/platform-tools/adb.exe'
   ```
   Adjust path based on where adb.exe was actually found in step 1.

3. Source ~/.zshrc and verify:
   ```bash
   source ~/.zshrc
   adb version
   ```
   Should print "Android Debug Bridge version X.X.X"

4. Determine Android version on connected device:
   ```bash
   adb kill-server && adb start-server
   adb devices
   adb shell getprop ro.build.version.release
   ```
   If `adb devices` shows no device, create a checkpoint asking user to connect phone with USB debugging enabled.
   Record the Android version — it determines the deployment method in step 5.

5. Package the verification test project as a .love file:
   ```bash
   cd /mnt/c/Users/Trynda/Desktop/Dev/Lua/projects/00-test
   zip -r /mnt/c/Users/Trynda/Desktop/Dev/Lua/test-game.love . -x "*.git*"
   ```

6. Deploy to Android device based on version:
   - **Android 10+** (version >= 10): Use lovegame folder method:
     ```bash
     adb push /mnt/c/Users/Trynda/Desktop/Dev/Lua/test-game.love "/sdcard/Android/data/org.love2d.android/files/games/lovegame/test-game.love"
     ```
     Note: User will need to manually open LOVE for Android app on phone.

   - **Android 9 and earlier** (version < 10): Use intent launch method:
     ```bash
     adb push /mnt/c/Users/Trynda/Desktop/Dev/Lua/test-game.love /sdcard/test-game.love
     adb shell am start -S -n "org.love2d.android/.GameActivity" -d "file:///sdcard/test-game.love"
     ```

7. Clean up the .love file after deployment:
   ```bash
   rm /mnt/c/Users/Trynda/Desktop/Dev/Lua/test-game.love
   ```

IMPORTANT: If adb push succeeds but the lovegame folder doesn't exist on the device, the LOVE for Android app may need to be launched once first to create it. Try launching the app manually, then retry the push.
  </action>
